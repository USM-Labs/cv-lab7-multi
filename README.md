- создана конфигурация nginx для связки с PHP-FPM;

## Краткая документация

- `docker-compose.yml` - описание трех сервисов, сети и volume.
- `nginx/default.conf` - конфигурация nginx для PHP.
- `mysql.env` - переменные окружения MySQL.
- `app.env` - переменная `APP_VERSION` для frontend и backend.
- `mounts/site/index.php` - демонстрационная PHP-страница.

## Безопасность

Пароли базы данных вынесены в файл `mysql.env`, что удобно для учебной среды, но в реальных проектах такой файл не должен попадать в публичный репозиторий. Для production-окружения лучше использовать секреты CI/CD или Docker Secrets.

## Ответы на контрольные вопросы

1. В каком порядке запускаются контейнеры?

   Docker Compose создает сеть и volume, затем запускает сервисы. В данном файле явной зависимости `depends_on` нет, поэтому порядок запуска не гарантирует готовность приложения. На практике сначала поднимается `database`, затем `backend` и `frontend`, но готовность БД нужно проверять отдельно.

2. Где хранятся данные базы данных?

   Данные MySQL хранятся в именованном volume `db_data`, который монтируется в `/var/lib/mysql`.

3. Как называются контейнеры проекта?

   При типовом запуске они называются `containers07-frontend-1`, `containers07-backend-1` и `containers07-database-1`.

4. Как добавить файл `app.env` с переменной `APP_VERSION` для `backend` и `frontend`?

   Нужно создать файл `app.env`:

   ```text
   APP_VERSION=1.0.0
   ```

   Затем добавить его в секцию `env_file` сервисов `frontend` и `backend`.
